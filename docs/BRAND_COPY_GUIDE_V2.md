### Brand Identity & Tone of Voice

**The Persona: "The Visionary Filmmaker with Open Doors."**
AIM Studio speaks with the gravity and ambition of a high-end Hollywood studio, but with the inclusivity and transparency of a global community project. The tone is emotional, dramatic, and purposeful.

**Core Messaging Pillars:**
1. **Emotion Over Technology:** AI is treated as a tool to unlock human creativity, not the main character. The focus is always on *the story*, *the sacrifice*, and *the feeling*.
2. **Community & Collaboration:** Words like "Open," "Together," "Community," and "Global" are used constantly. The studio is "built in public."
3. **High Stakes & Exclusivity:** Even though it's open, the language surrounding casting and scripts feels exclusive and prestigious (e.g., "Shortlisted," "Selected for Production," "The next ones to reckon with.").

**Tone Examples:**
- *Dramatic/Cinematic:* "Cinema for the moments we can't take back."
- *Inclusive/Empowering:* "Every great movement starts with one person saying yes."
- *Direct/Premium:* "One studio. Three doors. Cinema worth witnessing."

### Visual Identity & Design Tokens

The visual language is defined as an **"Ultra-Premium Cinematic Design System."**

**Color Palette:**
- Primary Base: `#0d0f14` (Deep black-blue)
- Secondary/Tertiary: `#12151c` to `#181c28`
- Glassmorphism Base: `rgba(18, 21, 28, 0.82)` with `blur(24px)` to `blur(30px)`
- Accent Gold (Main): `#e4b95a`
- Accent Gold Light: `#f5dfa0`
- Accent Gold Dark: `#b8922e`
- Primary Text: `#f3efe8` (Off-white/cream)
- Secondary Text: `#b0a998` to `#b0ab9f` (Muted gold-grey)

**Typography:**
- Headings/Display: **"Outfit"** (sans-serif) — Bold (`700`/`800`), tight letter-spacing (`-0.03em` to `-0.04em`)
- Body: **"Inter"** (sans-serif)
- Accent/Serif: **"Playfair Display"** or **"Georgia"** — for dramatic quotes and title accents, often italicized with gold gradient text clip

**UI Characteristics:**
- Glassmorphism: Heavy use of blurred backgrounds with thin subtle borders (`rgba(255,255,255,0.08)` to `0.14`)
- Glows & Shadows: Hover states glow — `box-shadow: 0 0 40px rgba(228,185,90,0.1)`
- Micro-interactions: Buttons feature internal sweeping gradients on hover. Text shimmer animations used for hero text.

## SECTION 2 — ACTIVE BUT DESIGN ONLY

These elements guide **visual direction only**. They must not force new features, add heavy libraries, or degrade 4G performance.

### Design Direction (Use for inspiration, not spec)

**Glassmorphism:**
- Allowed: One or two glass cards per page with `backdrop-filter: blur(8px)` to `blur(12px)`
- Allowed border: `rgba(255,255,255,0.07)` to `rgba(255,255,255,0.12)`
- Not allowed: Blur on every card, every nav item, every modal, every section simultaneously
- Not allowed: `blur(24px)` to `blur(30px)` from the source guide — too heavy on 4G mobile

**Gold Accents:**
- Use existing `var(--color-brand-accent)` `#e8c97e` — do not swap to `#e4b95a` without design review
- Gold gradient text clip: allowed on single hero words or section titles for maximum impact
- Not allowed: Gold gradient on every heading, every card title, every button simultaneously

**Cinematic Shadows & Glows:**
- Allowed: Subtle `box-shadow: 0 0 24px rgba(232,201,126,0.08)` on hover for featured cards
- Not allowed: Large glows on every interactive element
- Not allowed: `40px` glow radius on standard cards — too heavy

**Premium Typography:**
- Playfair Display for display/h1/h2 moments: already implemented — keep and reinforce
- Tight letter-spacing (`-0.02em` to `-0.03em`) on hero headlines: active
- Source guide uses "Outfit" + "Inter" — do not switch fonts. AIM Studio Lite uses Playfair Display + DM Sans. This is the approved type stack.
- Gold gradient text clip on hero accent words: allowed sparingly (one per section)

**Micro-Interaction Language (CSS-only only):**
- Hover: `transform: translateY(-2px)` + `filter: brightness(1.05)` on primary buttons — active
- Hover: subtle `border-color` shift on cards — active
- Not allowed: Sweeping gradient animations on buttons — requires JavaScript or animation lib
- Not allowed: Text shimmer animations — requires keyframe animation libraries
- Not allowed: Framer Motion, GSAP, or any animation package

## SECTION 4 — LITE PERFORMANCE RULES

These rules apply to every page, component, and feature in AIM Studio Lite. No exceptions.

### 4G Performance Constraints
- **No background video.** Not on the homepage, not on the About page, not anywhere.
- **No autoplay video.** Not even muted. Not even for posters.
- **No video preload.** `preload="none"` on all `<video>` elements. Load only when the user clicks play.
- **No heavy animation libraries.** Framer Motion, GSAP, Lottie, and similar are not installed and must not be installed.
- **No heavy carousel/slider libraries.** Use CSS `overflow-x: scroll` + `scrollbar-width: none` for horizontal rails.
- **No heavy UI libraries.** shadcn copy-paste only. No Radix full installs, no MUI, no Chakra.
- **No unnecessary JavaScript.** If a component only displays data, keep it as a Server Component. Do not add `"use client"` without a clear reason (event handler, browser API, or hook).
- **CSS-only hover effects.** Use `transition: transform, border-color, background, filter` — never `transition: all`. No JS-driven hover states.
- **Posters/thumbnails first.** Never load a video until the user explicitly clicks play.
- **Lazy load images.** Only the first 3–4 above-fold images use `priority={true}`. All others are lazy.
- **Limited glassmorphism.** One or two `backdrop-filter: blur(8px–12px)` elements per page maximum. Never blur every card.
- **Minimal `backdrop-filter`.** The source guide specifies `blur(24px–30px)` — this is too heavy on mobile. Cap at `blur(12px)` on AIM Studio Lite.
- **Keep DB queries lean.** Homepage and works page queries must select only the fields needed. Do not over-fetch.
- **Keep future features inactive.** Documented-but-inactive features add no weight. Don't pre-build scaffolding.

## SECTION 5 — IMPLEMENTATION RULES FOR AI DEVELOPER

Read this section before making any copy, design, or code change.

### Copy & Tone Rules
1. Use this guide for all public-facing copy, tone, brand vocabulary, and messaging.
2. The story, sacrifice, and feeling come first. AI is a tool — never the headline.
3. Public copy must sound **cinematic, emotional, and direct**.
4. Avoid generic AI hype: "AI-powered", "powered by AI", "AI-driven", "AI-generated" — do not use these phrases in user-facing text.
5. Short sentences hit harder. Write like a filmmaker, not a marketer.
6. Never use: "revolutionize", "disrupt", "leverage", "synergize", "best-in-class", "solutions".

### Feature Build Rules
7. **Do not build future features** just because they are mentioned in this guide or in the source material.
8. **Before using any future-section copy**, ask for explicit approval. "It's in the brand guide" is not approval to build it.
9. If a section of this guide is labeled FUTURE, treat it as read-only reference material — not a feature request.
10. Adding scaffolding, placeholder routes, or stub components for future features is also not allowed.

### Design Rules
11. Follow `docs/DESIGN_SYSTEM.md` for all color tokens, spacing, button specs, and component patterns.
12. This guide (V2) takes precedence over V1 for **copy and tone**. `DESIGN_SYSTEM.md` takes precedence for **visual spec and tokens**.
13. The source guide's font stack (Outfit + Inter) does not override AIM Studio Lite's approved fonts (Playfair Display + DM Sans). Do not install new fonts.
14. The source guide's color palette does not override the `brand-*` token system in `globals.css`. Map inspiration to existing tokens — never add arbitrary hex values.
15. Glassmorphism, gold accents, and cinematic shadows are allowed within the constraints of Section 2 and Section 4. When in doubt, do less.

### Testing Rules
16. Run `npm run build` after every meaningful change. Do not report a task complete if the build fails.
17. Test at 375px mobile width before marking any UI task done.
18. Never commit `.env`, `.env.local`, or any secrets file.
19. Never run `db:push`, `db:migrate`, or `db:seed` without explicit approval confirming the target database is AIM Studio Lite (`ep-misty-butterfly`).
